# StudyFlow - プロダクト要件定義書（PRD）

## 1. プロダクトビジョン

**中学受験の「リベンジノート」をデジタル化し、塾なし〜塾併用の家庭学習を親子で回せるツール**

### 1.1 ポジショニング

| 軸 | 既存プレイヤー | StudyFlow |
|---|---|---|
| 学習時間記録 | Studyplus（中高生〜社会人向け。小学生非対応） | ○ 小学生＋保護者前提 |
| 問題演習 | 一問一答系アプリ多数（コンテンツ依存） | × コンテンツは持たない |
| 塾向けLMS | Studyplus for School、atama+（B2B、塾経由） | × 塾は使わない |
| リベンジノート | 紙のコピー＋ノート（主流） | ◎ **ここがコア** |
| 受験スケジュール管理 | 紙の手帳、汎用カレンダー | △ Phase 2以降 |

**→ 「間違えた問題の記録→反復→定着」に特化。コンテンツではなく"学習プロセス管理"で勝負する。**

### 1.2 メインターゲット

```
偏差値50前後のボリュームゾーン × 塾なし〜中規模塾
├── 塾なし自宅学習派：コスパ重視、市販テキスト＋四谷大塚の予習シリーズ等
├── 中小塾＋家庭学習併用派：塾の宿題管理に困っている
└── 共通ペイン：「何を」「いつ」やり直すべきかが分からない
```

SAPIX/早稲アカ上位層は塾のシステムに囲い込まれており参入困難。
偏差値50帯（受験者の最大ボリューム）は塾に年間100万円超は出せないが、
月額1,000〜2,000円のツールには投資する層。

## 2. 技術アーキテクチャ

### 2.1 基本構成

```
[iPad / PC ブラウザ]
     │
     ├── Netlify (静的ホスティング)
     │     └── index.html（SPA、全ロジック内包）
     │
     └── データ永続化
           ├── ① localStorage（デフォルト、単一デバイス）
           ├── ② JSON エクスポート/インポート（手動バックアップ）
           └── ③ iCloud共有フォルダ経由の自動同期（親子共有）
```

### 2.2 データ同期設計（iCloud共有フォルダ案）

**前提**: ブラウザアプリからiCloudに直接書き込む標準APIは存在しない。
現実的な実装パターンは以下：

| 方式 | 実現性 | UX | 備考 |
|---|---|---|---|
| **A. JSON手動エクスポート→iCloud Driveに保存** | ◎ | △ | `<a download>` でJSONを落とし、iCloud Driveに手動配置。起動時にファイル選択で読み込み |
| **B. File System Access API** | △ | ○ | Chromium系のみ。Safari非対応のため iPad では使えない |
| **C. ショートカットApp連携** | ○ | ○ | iOSショートカットでJSON保存を自動化。ユーザー側の設定が必要 |
| **D. Supabase / Firebase** | ◎ | ◎ | クラウドDB。親子アカウント共有が最もスムーズだが、ランニングコストあり |

**推奨**: Phase 1はA（手動エクスポート/インポート）でMVP検証。
ユーザーが増えたらPhase 2でDを導入し月額課金の根拠にする。

## 4. UI/UX 設計方針

### 4.1 デザイン原則

| 原則 | 理由 |
|---|---|
| **子ども（小4-6）が一人で使える** | 親がいない時間に自習できることが価値 |
| **親は"見る"がメイン、"入力"は最小限** | 共働き家庭は夜しか見られない。ダッシュボードで把握できる |
| **紙のノートの延長線** | デジタルネイティブでも紙の問題集は使う。撮影→貼付の流れを徹底的に楽に |
| **シンプル > 多機能** | 塾のLMSのような複雑さは不要。Notionではなくメモ帳感覚 |

## 5. 収益化ロードマップ

### Phase 1: 無料配布＋NOTE収益（0〜3ヶ月）

- Netlifyで無料ホスティング
- NOTEで開発ストーリー＋使い方記事を連載
- 記事自体の有料販売（¥300〜500）または投げ銭
- **目的**: PMF検証、ユーザーの声を集める

### Phase 2: フリーミアム（3〜6ヶ月）

- 基本機能は無料継続（問題登録、正誤記録、忘却曲線）
- 有料機能（月額¥980）：
  - 親ダッシュボード
  - クラウド同期（親子共有）
  - 学習カレンダー
  - データエクスポート（CSV/PDF）
- **決済**: Stripe or lemonsqueezy（個人開発向け）

### Phase 3: B2B2C（6ヶ月〜）

- 中小塾への卸売り：塾が保護者に推奨→保護者が契約
- 塾向け管理画面（Studyplus for School的なポジション、ただし中学受験特化）
- **単価**: 塾あたり月額¥5,000〜¥30,000（生徒数連動）

### 収益試算（保守的）

```
Phase 2: 
  有料ユーザー 100世帯 × ¥980/月 = ¥98,000/月（¥1.17M/年）

Phase 3:
  個人ユーザー 500世帯 × ¥980/月 = ¥490,000/月
  塾契約 10塾 × ¥15,000/月 = ¥150,000/月
  合計: ¥640,000/月（¥7.68M/年）
```

## 6. 競合との差別化まとめ

| 比較軸 | Studyplus | 一問一答系アプリ | 紙のリベンジノート | **StudyFlow** |
|---|---|---|---|---|
| 対象学年 | 中学〜社会人 | 小学〜中学 | 全学年 | **小4〜6（中学受験特化）** |
| コンテンツ | なし | アプリ内蔵 | 塾テキストのコピー | **ユーザーが自分の教材を登録** |
| 問題のやり直し | × | ○（アプリ内のみ） | ◎（手書き） | **◎（手書き＋デジタル記録）** |
| 忘却曲線 | × | △ | × | **◎** |
| 保護者共有 | × | × | △（ノートを見る） | **◎（ダッシュボード）** |
| データ蓄積 | ○（時間のみ） | △ | ×（紙は検索不可） | **◎（正誤推移、弱点分析）** |
| 導入コスト | 無料 | 無料〜¥4,000 | ¥0（紙代のみ） | **無料〜¥980/月** |
